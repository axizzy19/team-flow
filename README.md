# Проект: Веб-приложение для управления задачами, сроками и ресурсами в команде «TeamFlow»

### Функциональные требования:
Возможность создание проектов с названием, описанием, сроками. \
Создание и распределение задач внутри проекта среди членов команды с указанием дедлайна, статуса и приоритета. \
Возможность редактировать существующие задачи. \
Система комментариев внутри каждой задачи. \
Функция уведомления об изменениях в задачах и приближающихся дедлайнах. \
Отображение прогресса по задачам с помощью Канбан-доски. \
Настройка прав доступа. 

### Нефункциональные требования:
Система должна обеспечивать приемлемое время отклика (не более 5 секунд) при работе до 20 одновременных пользователей. \
Доступ к системе должен осуществляться через аутентификацию. Данные должны передаваться по защищенному соединению HTTPS. \
Веб-приложение должно корректно работать в основных браузерах (Chrome, Firefox, Safari, Edge) и быть адаптивным для мобильных устройств.

### Описание взаимодействия пользователя с системой:
Руководитель создает новый проект, заполняет его основные параметры и приглашает участников по email. \
Участник команды входит в систему, видит назначенные ему задачи, меняет их статус и оставляет комментарии о прогрессе. \
Система синхронизирует изменения в режиме реального времени. \
Руководитель отслеживает прогресс проекта.
